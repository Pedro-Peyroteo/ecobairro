| #   | Método   | Rota                    | Descrição                              | Auth                      | Fluxo                                                                        |
| --- | -------- | ----------------------- | -------------------------------------- | ------------------------- | ---------------------------------------------------------------------------- |
| A1  | `POST`   | `/auth/register`        | Registo de novo cidadão                | Público                   | NestJS → PG write users (role=CIDADAO) + cidadao_perfis → email queue BullMQ |
| A2  | `POST`   | `/auth/login`           | Login com email + password             | Público                   | NestJS → PG read users (verify hash) → JWT + Redis session                   |
| A3  | `POST`   | `/auth/refresh`         | Renovar access token via refresh token | Refresh token             | NestJS → Redis session check                                                 |
| A4  | `POST`   | `/auth/logout`          | Revogar sessão                         | Qualquer role autenticado | NestJS → Redis del session                                                   |
| A5  | `POST`   | `/auth/forgot-password` | Solicitar reset por email              | Público                   | NestJS → Redis store token (TTL 1h) → BullMQ email                           |
| A6  | `POST`   | `/auth/reset-password`  | Aplicar nova password via token        | Público + reset token     | NestJS → Redis verify token → PG write users → Redis del                     |
| A7  | `POST`   | `/auth/verify-email`    | Confirmar email via token              | Público + verify token    | NestJS → Redis verify → PG write users.email_verified=true                   |
| A8  | `POST`   | `/auth/2fa/enable`      | Iniciar activação TOTP/SMS             | Qualquer role autenticado | NestJS → gera secret → Redis temp (TTL 5min)                                 |
| A9  | `POST`   | `/auth/2fa/confirm`     | Confirmar código e activar 2FA         | Qualquer role autenticado | NestJS → Redis verify → PG write users.2fa_enabled=true                      |
| A10 | `DELETE` | `/auth/2fa`             | Desactivar 2FA                         | Qualquer role autenticado | NestJS → PG write users → Redis invalidar                                    |
| A11 | `POST`   | `/auth/2fa/verify`      | Validar código 2FA durante login       | Público + pre-auth token  | NestJS → verifica TOTP/SMS → emite JWT completo                              |

**Corpo de A1 (Registo de cidadão):**

```
{
  email: string (required),
  password: string (required, min 10 chars),
  phone: string (optional),
  rgpd_accepted: boolean (required, must be true)
}
```

**Fluxo interno de A1:**
1. Cria registo em `users` com `role = 'CIDADAO'`
2. Cria registo em `cidadao_perfis` com `user_id = users.id` e `rgpd_accepted = true`
3. Ambos numa transação PostgreSQL
4. Envia email de verificação via BullMQ

Nota: O registo é exclusivo para cidadãos. Operadores são criados pelo admin via `POST /admin/operadores` (ver PA5 em [[1.2 Perfil do Cidadão (CRUD)]]).

**Resposta de A2 (Login com sucesso):**

```
{
  access_token: string (JWT, TTL 15min),
  refresh_token: string (opaque, TTL 7d),
  requires_2fa: boolean,
  pre_auth_token: string | null  (se requires_2fa=true)
}
```

Nota: O JWT payload inclui `{ sub: users.id, role: users.role }`. O login funciona para qualquer role — a tabela consultada é sempre `users`.
