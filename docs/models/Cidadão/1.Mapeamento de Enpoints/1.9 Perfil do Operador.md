#### Perfil do Operador (self-service)

Endpoints para operadores (OPERADOR_VEOLIA, TECNICO_AUTARQUIA, TECNICO_CCDR, ADMIN) gerirem o seu próprio perfil.

| #   | Método | Rota              | Descrição                                    | Auth                                                              | Fluxo                                                              |
| --- | ------ | ----------------- | -------------------------------------------- | ----------------------------------------------------------------- | ------------------------------------------------------------------ |
| PO1 | `GET`  | `/operadores/me`  | Obter perfil próprio do operador             | OPERADOR_VEOLIA, TECNICO_AUTARQUIA, TECNICO_CCDR, ADMIN          | NestJS → Redis hit (operador:profile:{id}) ou PG réplica (JOIN users + operador_perfis) |
| PO2 | `PUT`  | `/operadores/me`  | Atualizar campos editáveis (cargo, phone)    | OPERADOR_VEOLIA, TECNICO_AUTARQUIA, TECNICO_CCDR, ADMIN          | NestJS → PG write users (phone) + operador_perfis (cargo) → NOTIFY operador_updated → Redis invalidar |

**Resposta de PO1:**

```
{
  id: uuid (users.id),
  email: string,
  phone: string | null,
  role: string,
  email_verified: boolean,
  nome_completo: string,
  entidade_empregadora: string,
  cargo: string | null,
  zonas_responsabilidade: [
    { id: uuid, nome: string }
  ],
  criado_em: ISO8601
}
```

**Corpo de PO2 (Atualizar perfil operador):**

```
{
  phone: string (optional),
  cargo: string (optional)
}
```

Nota: Os campos `nome_completo`, `entidade_empregadora`, `zonas_responsabilidade` e `role` só podem ser alterados pelo ADMIN via `PUT /admin/operadores/:id` (PA8). O operador pode apenas editar `phone` (em `users`) e `cargo` (em `operador_perfis`).

**Cache Redis:**

```
operador:profile:{user_id}  TTL: 30min
  Campos: email, phone, role, nome_completo, entidade_empregadora,
          cargo, zonas_responsabilidade (com nomes resolvidos)
  Invalidação: NOTIFY operador_updated → DEL operador:profile:{id}
```
