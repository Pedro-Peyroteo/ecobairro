**Notas de design:**
- Os campos `nif_cifrado` e `morada_cifrada` são cifrados AES-256 pela camada de aplicação antes de escrever no PostgreSQL. O banco nunca vê o valor em claro (RNF-PRIV-01).
- A busca por NIF passa pela camada app que calcula HMAC-SHA256(nif, chave_secreta_app) e compara com `nif_hash`.
- O soft delete do cidadão é controlado pela coluna `users.eliminado_em` — esta tabela não tem `eliminado_em` próprio, sendo cascadeada pelo ON DELETE CASCADE ou pelo job BullMQ de anonimização que limpa os campos PII.
- Dados PII desta tabela **NUNCA** são cacheados em Redis (RNF-PRIV-01/04).
