### Índices estratégicos — tabelas de identidade e perfis

#### Tabela `users`

```
┌──────────────────────────────────────┬────────────────┬──────────────────────┐
│ Coluna(s)                            │ Tipo           │ Uso                  │
├──────────────────────────────────────┼────────────────┼──────────────────────┤
│ (email)                              │ UNIQUE BTREE   │ login lookup         │
│ (eliminado_em)                       │ PARTIAL BTREE  │ archiving queries    │
│                                      │ WHERE NOT NULL │                      │
│ (role)                               │ BTREE          │ filtros admin por    │
│                                      │                │ tipo de utilizador   │
└──────────────────────────────────────┴────────────────┴──────────────────────┘
```

#### Tabela `cidadao_perfis`

```
┌──────────────────────────────────────┬────────────────┬──────────────────────┐
│ Coluna(s)                            │ Tipo           │ Uso                  │
├──────────────────────────────────────┼────────────────┼──────────────────────┤
│ (user_id)                            │ UNIQUE BTREE   │ join rápido com users│
│ (nif_hash)                           │ BTREE          │ pesquisa admin NIF   │
│                                      │                │ sem decifrar todos   │
│ (nome_completo)                      │ GIN pg_trgm    │ full-text ILIKE      │
│                                      │                │ '%nome%' em ms       │
│ (localizacao_ultima)                 │ GIST           │ ST_DWithin proximity │
│                                      │                │ R-tree interno       │
│ (zona_residencia_id)                 │ BTREE PARTIAL  │ cidadãos por zona    │
│                                      │ WHERE user_id  │ (filtra soft-deleted │
│                                      │ NOT IN soft-   │  via join users)     │
│                                      │ deleted        │                      │
└──────────────────────────────────────┴────────────────┴──────────────────────┘
```

#### Nota sobre `nif_hash`

Coluna nif_hash explicada: O NIF não é pesquisável no valor cifrado. Cria-se uma coluna adicional `cidadao_perfis.nif_hash` VARCHAR(64) que armazena HMAC-SHA256(nif, chave_secreta_app). A chave secreta é um secret Swarm — nunca na base de dados.
Assim o admin pode pesquisar o cidadão pelo NIF sem expor o NIF em claro nem decifrar todos os registos.
