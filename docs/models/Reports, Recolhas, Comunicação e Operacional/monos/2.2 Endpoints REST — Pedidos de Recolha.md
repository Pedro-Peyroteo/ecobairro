| #   | Método   | Rota                                 | Descrição                                               | Auth                   | Fluxo                                                     |
| --- | -------- | ------------------------------------ | ------------------------------------------------------- | ---------------------- | --------------------------------------------------------- |
| M1  | `POST`   | `/recolhas/monos`                    | Submeter pedido RF-14 — gera número de pedido           | CIDADAO                | NestJS → PG write → BullMQ → operador responsável         |
| M2  | `GET`    | `/cidadaos/me/recolhas`              | Listar pedidos do cidadão com estado                    | CIDADAO                | NestJS → PG réplica paginado                              |
| M3  | `GET`    | `/cidadaos/me/recolhas/:id`          | Detalhe do pedido                                       | CIDADAO                | NestJS → PG réplica                                       |
| M4  | `DELETE` | `/cidadaos/me/recolhas/:id`          | Cancelar pedido (só se PENDENTE)                        | CIDADAO                | NestJS → valida estado → PG write CANCELADO               |
| M5  | `GET`    | `/operacional/recolhas`              | Listar pedidos para gestão `?zona_id=&estado=&de=&ate=` | OPERADOR_VEOLIA, ADMIN | NestJS → PG réplica                                       |
| M6  | `PATCH`  | `/operacional/recolhas/:id/agendar`  | Agendar data de recolha e atribuir operador             | OPERADOR_VEOLIA, ADMIN | NestJS → PG write AGENDADO + data → NOTIFY → push cidadão |
| M7  | `PATCH`  | `/operacional/recolhas/:id/concluir` | Marcar como concluído                                   | OPERADOR_VEOLIA, ADMIN | NestJS → PG write CONCLUIDO → NOTIFY                      |
| M8  | `PATCH`  | `/operacional/recolhas/:id/rejeitar` | Rejeitar com motivo                                     | OPERADOR_VEOLIA, ADMIN | NestJS → PG write REJEITADO + motivo → NOTIFY             |
