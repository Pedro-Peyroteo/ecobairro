Cada transição de estado gera uma linha — audit trail completo e visível ao cidadão (RF-11).

```
┌─────────────────────────────────────────────────────────────────────────────┐
│  tabela: reports_timeline                                                   │
├──────────────────────────┬──────────────────────┬───────────────────────────┤
│ coluna                   │ tipo                 │ constraints / notas       │
├──────────────────────────┼──────────────────────┼───────────────────────────┤
│ id                       │ UUID                 │ PK                        │
│ report_id                │ UUID                 │ NOT NULL                  │
│                          │                      │ FK → reports(id)          │
│ estado_anterior          │ VARCHAR(20)          │ nullable                  │
│                          │                      │ NULL = criação inicial    │
│ estado_novo              │ VARCHAR(20)          │ NOT NULL                  │
│ nota                     │ TEXT                 │ nullable                  │
│                          │                      │ mensagem visível ao       │
│                          │                      │ cidadão (RF-11)           │
│ alterado_por             │ UUID                 │ nullable                  │
│                          │                      │ FK → users(id)            │
│                          │                      │ NULL = sistema automático │
│ alterado_em              │ TIMESTAMPTZ          │ NOT NULL, DEFAULT now()   │
└──────────────────────────┴──────────────────────┴───────────────────────────┘

Índice: (report_id, alterado_em ASC) — timeline em ordem cronológica
```
