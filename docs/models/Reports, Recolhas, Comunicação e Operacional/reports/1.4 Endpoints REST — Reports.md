### Cidadão

|#|Método|Rota|Descrição|Auth|Fluxo|
|---|---|---|---|---|---|
|R1|`POST`|`/reports`|Criar report georreferenciado RF-08|CIDADAO|NestJS → verifica antispam Redis → valida zona → PG write → BullMQ triagem|
|R2|`GET`|`/reports/proximos`|Reports abertos próximos de um ponto `?lat=&lng=&raio=`|CIDADAO|FastAPI → PG réplica ST_DWithin|
|R3|`GET`|`/cidadaos/me/reports`|Reports criados pelo cidadão autenticado|CIDADAO|NestJS → PG réplica paginado|
|R4|`GET`|`/reports/:id`|Detalhe + timeline visível ao cidadão|CIDADAO|NestJS → PG réplica join timeline|
|R5|`POST`|`/reports/:id/subscrever`|Subscrever report existente RF-12|CIDADAO|NestJS → INSERT subscricoes → UPDATE num_subscricoes|
|R6|`DELETE`|`/reports/:id/subscrever`|Cancelar subscrição|CIDADAO|NestJS → DELETE subscricoes → UPDATE num_subscricoes|
|R7|`GET`|`/reports/quota`|Ver quota anti-spam restante por zona RF-09|CIDADAO|NestJS → Redis antispam keys|
|R8|`GET`|`/reports/duplicados`|Detectar possíveis duplicados antes de submeter `?lat=&lng=&categoria=`|CIDADAO|FastAPI → PG réplica ST_DWithin + categoria + últimos 7 dias|

### Operacional

|#|Método|Rota|Descrição|Auth|Fluxo|
|---|---|---|---|---|---|
|R9|`GET`|`/operacional/reports`|Listar reports com filtros para triagem RF-10|OPERADOR_VEOLIA, TECNICO_AUTARQUIA, ADMIN|NestJS → PG réplica|
|R10|`PATCH`|`/operacional/reports/:id/estado`|Avançar estado com nota RF-11|OPERADOR_VEOLIA, TECNICO_AUTARQUIA, ADMIN|NestJS → PG write reports + INSERT timeline → NOTIFY|
|R11|`PATCH`|`/operacional/reports/:id/atribuir`|Atribuir report a operador|TECNICO_AUTARQUIA, ADMIN|NestJS → PG write atribuido_a|
|R12|`GET`|`/operacional/reports/kpis`|Tempo médio resolução, por categoria/zona RF-23|TECNICO_AUTARQUIA, ADMIN|FastAPI → Redis 15min ou PG réplica|

**Corpo de R1 (criar report):**

```
{
  categoria: "ECOPONTO_CHEIO" | "ECOPONTO_PARTIDO" | "DEPOSICAO_ILEGAL"
           | "ODORES" | "GAIVOTAS_ORGANICOS" | "VANDALISMO" | "OUTRO",
  descricao: string (optional),
  localizacao: { lat: float, lng: float } (required),
  localizacao_manual: boolean (default false),
  ecoponto_id: uuid (optional),
  foto_base64: string (optional — NestJS faz upload para MinIO async)
}
```

**Pipeline de R1:**

```
POST /reports
  │
  ├── 1. Redis INCR antispam:report:{cidadao_id}:{zona_id}
  │         SE >= limite_antispam_reports → 429 com mensagem RF-09
  │
  ├── 2. ST_Within(localizacao, zonas) → determina zona_id
  │         SE ponto fora de qualquer zona → 422 zona não encontrada
  │
  ├── 3. ST_DWithin(localizacao, reports recentes, raio_dedup)
  │         SE encontrar duplicado → 200 com sugestão de subscrever RF-12
  │         (não bloqueia — cidadão pode continuar)
  │
  ├── 4. INSERT reports → PG primário
  │
  ├── 5. INSERT reports_timeline (estado_anterior=NULL, estado_novo='RECEBIDO')
  │
  ├── 6. BullMQ job: 'reports.triagem'
  │         → regra categoria+zona → define entidade_responsavel RF-10
  │         → NOTIFY report_criado → Redis invalidar cache zona
  │
  └── 7. BullMQ job: 'reports.foto-upload' (se foto enviada)
            → upload MinIO → UPDATE reports.foto_url, foto_thumbnail_url
```

**Corpo de R10 (avançar estado):**

```
{
  estado: "EM_ANALISE" | "EM_RESOLUCAO" | "RESOLVIDO" | "REJEITADO",
  nota: string (optional — visível ao cidadão),
  motivo_rejeicao: string (obrigatório se estado = 'REJEITADO')
}
```

Após R10, o NestJS emite NOTIFY `report_estado_updated` que o WS gateway captura para enviar notificação push/email ao cidadão (RF-11 e RF-16).

**Cache Redis — Reports:**

