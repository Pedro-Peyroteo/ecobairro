**Resposta de OP3 (fila de prioridades):**

```
[
  {
    ecoponto_id: uuid,
    nome: string,
    localizacao: { lat, lng },
    zona_nome: string,
    nivel_enchimento: integer,
    estado: string,
    ultima_leitura_em: ISO8601,
    minutos_sem_leitura: integer,
    score_prioridade: float,   ← calculado: nivel * peso_nivel
                                  + minutos_offline * peso_tempo
    alertas_abertos: integer
  },
  ...
]
```

O `score_prioridade` é calculado no FastAPI com pesos configuráveis por zona. Ecopontos em `OFFLINE` há mais de 2× o `timeout_offline_min` têm penalização adicional de prioridade — não significa que estão cheios, mas que precisam de verificação.
