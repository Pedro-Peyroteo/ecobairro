```
┌─────────────────────────────────────────────────────────────────────────────┐
│  tabela: ecoponto_estado_atual                                              │
│  1 linha por ecoponto — actualizada atomicamente por cada leitura IoT      │
├──────────────────────────┬──────────────────────┬───────────────────────────┤
│ coluna                   │ tipo                 │ constraints / notas       │
├──────────────────────────┼──────────────────────┼───────────────────────────┤
│ ecoponto_id              │ UUID                 │ PK                        │
│                          │                      │ FK → ecopontos(id)        │
│                          │                      │ ON DELETE CASCADE         │
│ dispositivo_id           │ UUID                 │ nullable                  │
│                          │                      │ FK → iot_dispositivos(id) │
│                          │                      │ NULL = sem sensor activo  │
├──────────────────────────┼──────────────────────┼───────────────────────────┤
│ nivel_enchimento         │ SMALLINT             │ nullable                  │
│                          │                      │ CHECK (0 <= v <= 100)     │
│ estado                   │ VARCHAR(20)          │ NOT NULL                  │
│                          │                      │ DEFAULT 'SEM_SENSOR'      │
│                          │                      │ CHECK IN (               │
│                          │                      │  'CHEIO',                 │
│                          │                      │  'DISPONIVEL',            │
│                          │                      │  'SEM_SENSOR',            │
│                          │                      │  'AVARIADO',              │
│                          │                      │  'MANUTENCAO',            │
│                          │                      │  'OFFLINE')               │
│ ultima_leitura_em        │ TIMESTAMPTZ          │ nullable                  │
│ leitura_valida           │ BOOLEAN              │ NOT NULL, DEFAULT true    │
│                          │                      │ false = último heartbeat  │
│                          │                      │ foi inválido              │
│ offline_desde            │ TIMESTAMPTZ          │ nullable                  │

│                          │                      │ estado = 'OFFLINE'        │

│ battery_pct              │ SMALLINT             │ nullable                  │

```

**Decisão — estado `OFFLINE`:** O RNF-CONF-02 exige que se o sensor estiver offline, o ecoponto apareça como "sem leitura recente" com data/hora. O estado `OFFLINE` na tabela `ecoponto_estado_atual` serve exactamente este propósito. Um job BullMQ periódico (`iot.offline-detector`) percorre dispositivos activos e marca como OFFLINE aqueles que não enviaram heartbeat há mais de `config.timeout_offline_min` minutos.
