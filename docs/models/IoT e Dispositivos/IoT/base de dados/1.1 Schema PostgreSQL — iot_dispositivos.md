**Decisão — `api_key_hash`:** Cada dispositivo IoT autentica-se com uma API key própria. A key nunca é guardada em claro — só o HMAC-SHA256. No momento de ingestão (endpoint E11), o NestJS recebe a key no header `X-Device-Key`, calcula o HMAC e compara com `api_key_hash`. Se coincidir, a leitura é aceite. A key é gerada no registo e mostrada uma única vez ao admin.

**Decisão — `config` em JSONB:** Parâmetros de calibração e thresholds variam por modelo de sensor e localização de instalação. O `calibracao_offset` corrige desvios sistemáticos do sensor específico. O `timeout_offline_min` define quando o dispositivo é considerado offline (default 30 min, mas pode ser maior para sensores com bateria em zonas rurais).
