| #   | Método  | Rota                                    | Descrição                                                        | Auth                                      | Fluxo                                                        |
| --- | ------- | --------------------------------------- | ---------------------------------------------------------------- | ----------------------------------------- | ------------------------------------------------------------ |
| D1  | `GET`   | `/iot/dispositivos`                     | Listar dispositivos com filtros `?estado=&ecoponto_id=&zona_id=` | OPERADOR_VEOLIA, TECNICO_AUTARQUIA, ADMIN | NestJS → PG réplica                                          |
| D2  | `GET`   | `/iot/dispositivos/:id`                 | Detalhe completo: estado, último heartbeat, config               | OPERADOR_VEOLIA, TECNICO_AUTARQUIA, ADMIN | NestJS → PG réplica                                          |
| D3  | `GET`   | `/iot/dispositivos/:id/leituras`        | Histórico de leituras paginado com filtro `?de=&ate=&validas=`   | TECNICO_AUTARQUIA, ADMIN                  | FastAPI → PG réplica `sensor_leituras` paginado              |
| D4  | `GET`   | `/iot/dispositivos/:id/alertas`         | Alertas deste dispositivo                                        | OPERADOR_VEOLIA, ADMIN                    | NestJS → PG réplica                                          |
| D5  | `POST`  | `/iot/dispositivos`                     | Registar novo dispositivo — devolve a API key uma única vez      | ADMIN                                     | NestJS → gera API key → guarda HMAC → PG write               |
| D6  | `PUT`   | `/iot/dispositivos/:id`                 | Actualizar metadados e configuração                              | TECNICO_AUTARQUIA, ADMIN                  | NestJS → PG write → Redis invalidar estado                   |
| D7  | `PATCH` | `/iot/dispositivos/:id/instalar`        | Instalar em ecoponto — cria registo histórico                    | TECNICO_AUTARQUIA, ADMIN                  | NestJS → PG write instalar + histórico + UPSERT estado_atual |
| D8  | `PATCH` | `/iot/dispositivos/:id/desinstalar`     | Retirar de ecoponto — ecoponto volta a SEM_SENSOR                | TECNICO_AUTARQUIA, ADMIN                  | NestJS → PG write desinstalar + histórico + estado_atual     |
| D9  | `PATCH` | `/iot/dispositivos/:id/estado`          | Alterar estado manual: MANUTENCAO, AVARIADO, DESCONTINUADO       | TECNICO_AUTARQUIA, ADMIN                  | NestJS → PG write + histórico                                |
| D10 | `POST`  | `/iot/dispositivos/:id/api-key/renovar` | Gerar nova API key — invalida a anterior                         | ADMIN                                     | NestJS → gera nova key → guarda HMAC → PG write              |
| D11 | `GET`   | `/iot/dispositivos/offline`             | Lista de dispositivos sem heartbeat há mais de X minutos         | OPERADOR_VEOLIA, ADMIN                    | NestJS → PG réplica WHERE ultimo_heartbeat < now() - timeout |

**Corpo de D5 (registar dispositivo):**

```
{
  device_id: string (required — ID hardware imutável),
  nome: string (required),
  tipo: "NIVEL_ULTRASONICO" | "NIVEL_INFRAVERMELHO" | "PESO"
       | "COMPACTACAO_SOLAR" | "MULTIPARAMETRO",
  fabricante: string (optional),
  modelo: string (optional),
  numero_serie: string (optional),
  protocolo_comunicacao: "HTTP" | "MQTT" | "LORAWAN" | "SIGFOX" | "NB_IOT",
  intervalo_leitura_seg: integer (default 300),
  config: {
    limiar_alerta_battery: integer (default 20),
    timeout_offline_min: integer (default 30),
    calibracao_offset: integer (default 0),
    altura_instalacao_cm: integer (default 0)
  }
}
```

**Resposta de D5 (única vez que a API key é exposta):**

```
{
  dispositivo_id: uuid,
  device_id: string,
  api_key: string  ← guardada APENAS nesta resposta, nunca mais acessível
                      instrução ao admin: guardar de imediato no hardware
}
```

**Corpo de D7 (instalar em ecoponto):**

```
{
  ecoponto_id: uuid (required),
  motivo: string (optional)
}
```

O NestJS executa em transacção:

1. Verifica que o ecoponto não tem outro dispositivo ATIVO do mesmo tipo
2. Actualiza `iot_dispositivos.ecoponto_id` e `instalado_em`
3. Muda `estado` para `ATIVO`
4. Insere linha em `iot_dispositivos_historico`
5. Actualiza `ecoponto_estado_atual.dispositivo_id`
6. Actualiza `ecopontos.tem_sensor = true`
